# Repository Foundation Design

## Intent

Build `awesome-openclaw-business-skills` as a public sample repository for enterprise AI adoption rather than a generic skill dump.

## Positioning

- Practical business skills
- Enterprise workflow orientation
- Public baseline plus custom enterprise layering
- Portfolio, learning asset, and consulting entry point in one repository

## Structure

- Root docs establish trust and make the README function like a product homepage.
- `skills/` contains five representative business workflow samples.
- `docs/` explains principles, onboarding, and enterprise framing.
- `.github/` sets contribution standards and quality expectations.
- `templates/` helps future expansion stay consistent.

## Release Shape

Start with three baseline skills that are easy to understand and commercially relevant:

- meeting-minutes
- client-followup
- weekly-report

## Quality Bar

- Every skill must be scannable.
- Every skill must show example input and output.
- Public content stays generic and reusable.
- Advanced enterprise customization remains intentionally outside the public baseline.
